**Limus** A modern langage and optimizing compiler. The name means mud, slime or muck in Latin and refer to [Physarum polycephalum](https://en.wikipedia.org/wiki/Physarum_polycephalum) a slime mold able to solve optimization problems. # Langage ## Ideas - The assume keyword allow the programmer to express it's own knowledge - Access the parity, carry and overflow flags without assembly - Native support for fixed-point - What you use is what you get (WYUIWYG) # Compiler ## Lex, Parse and Syntax Tree ### Parsing Expression Grammar #### Introduction Parsing expression grammar, or PEG, is a type of analytic formal grammar syntactically similar to [Extended Backus-Naur form](https://en.wikipedia.org/wiki/Extended_Backus%E2%80%93Naur_form) and [Regular Expression](https://en.wikipedia.org/wiki/Regular_expression). It was introduced by Bryan Ford in 2004 and is closely related to the family of [top-down parsing](https://en.wikipedia.org/wiki/Top-down_parsing) languages introduced in the early 1970s. #### Syntax !!! note The reference below describes the Cut operator a generalization of the If-then-else operator. |Operators|Syntax|Semantic|Type [#Hutton1996]| |---|---|---|---| |Nothing|$\epsilon$|Match nothing once|Unit| |Sequence|$e_1\ e_2$|Match $e_1$ then $e_2$|(typeOf($e_1$), typeOf($e_2$))| |Ordered choice|$e_1 / e_2$|Match $e_1$ first or $e_2$ then|sharedTypeOf($e_1$, $e_2$)| |Zero-or-more|$e^* \Leftrightarrow N \leftarrow e N / \epsilon$|Match $e$ zero or more time(s)|Option`<`Vec`<`typeOf($e$)`>>`| |One-or-more|$e^+ \Leftrightarrow ee^*$|Match $e$ once or more time(s)|Vec`<`typeOf($e$)`>`| |Optional|$e? \Leftrightarrow e / \epsilon$|Match $e$ if possible|Option`<`typeOf($e$)`>`| |And-predicate|&e|Look ahead if e match|Unit| |Not-predicate|!e|Look ahead if e doesn't match|Unit| |If-then-else [#Mizushima2010]|$e\ ?\ e_1 : e_2 \Leftrightarrow (e\ e_1) / (!e\ e_2)$|Match $e_1$ if $e$ succeed else $e_2$|(bool, sharedTypeOf($e_1$, $e_2$))| |Redirect|$e$ > func|Call func when $e$ match|returnTypeOf(func)| #### Example Calculator that recognize the basic four operations to non-negative integers with precedence handling. ***************************************** * .-------------------------------. * * | Product | * * | .-------------------. | * * | | Expr | | * * | .---. | .---. .---. | | * * | |Num| | |Num| |Num| | | * * | | 3 | x | ( | 2 | + | 1 | ) | | = 9 * * | '---' | '---' '---' | | * * | '-------------------' | * * '-------------------------------' * ***************************************** #### Advantages # Bibliography [#Hutton1996]: Graham Hutton and Erik Meijer. 1996.
